# Projects Tracker Design System Spec (Untitled UI Classic)

## Overview

Проект `Projects Tracker` следует канонической дизайн-системе **Untitled UI Classic (Slate)**. Стиль сочетает в себе плоский архитектурный минимализм и строгую корпоративную эстетику. 

Цель дизайн-системы — предоставить понятный, не отвлекающий интерфейс с высокой плотностью полезной информации для преподавателя и удобной интерактивной средой для учеников. Основной акцент делается на чистые плоские геометрические формы, сдержанную серую гамму в качестве базы и мощные акцентные цвета (фиолетовый, изумрудный, янтарный) для обозначения разделов, состояний и интерактивности.

---

## Colors

Дизайн-система поддерживает **4 цветовых пресета** в двух режимах (светлом и темном):
1. **Untitled Indigo / Lavender (amethyst-eclipse)** — дефолтный пресет с глубоким аметистовым акцентом.
2. **Untitled Emerald / Mint (cyber-emerald)** — пресет с чистым неоновым изумрудным акцентом.
3. **Untitled Amber / Copper (amber-core)** — пресет с теплым оранжево-янтарным свечением.
4. **Untitled Classic UI (untitled-classic)** — классическая корпоративная серая тема с благородными пурпурными деталями.

### 1. Light Mode (Светлая тема)
- **Фон страницы (`bg-light`):** `#FAFAFB` — чистый, прохладный светлый холст.
- **Вторичный фон (`bg-light-sec`):** `#F4F5F7` / `#F2F4F7` для боковых панелей и карточек.
- **Основной текст (`text-light`):** `#101828` — высококонтрастный графитовый/черный.
- **Границы (`border-light`):** `#EAECF0` — ультра-тонкие чистые светло-серые рамки.

### 2. Dark Mode (Темная тема)
- **Фон страницы (`bg-dark`):** `#0C111D` — истинный темный сапфир/уголь без синего или зеленого перекоса.
- **Вторичный фон (`bg-dark-sec`):** `#161B26` / `#1F242F` для выделения блоков.
- **Основной текст (`text-dark`):** `#F9FAFB` — чистый мягкий белый.
- **Границы (`border-dark`):** `#222939` — деликатные полупрозрачные темные разделители.

### 3. Interactive Colors (Связующие цвета)
- **Primary Brand Purple (`primary-brand`):** `#7F56D9` (Light) и `#9E77ED` (Dark) — ключевой интерактивный путеводитель для кнопок, фокусов ввода и активных состояний.
- **Contrast Text on Buttons:** Всегда **белый** (`#FFFFFF` / `0 0% 100%`) для первичных (default) и деструктивных (destructive) кнопок в обоих режимах (светлом и темном), исключая темный текст на ярких кнопках.

## Typography

Основная шрифтовая гарнитура проекта — **Inter**. 

### Правила и ограничения:
- **Шрифт без исключений:** Все интерактивные контролы, кнопки, поля ввода, таблицы и тексты принудительно используют класс `font-sans` для предотвращения сброса шрифтовой гарнитуры в браузерах.
- **Display-заголовки:** Используют плотное начертание `semibold` / `bold` с уменьшенным межбуквенным интервалом (`tracking-tight` / `-0.02em`).
- **Body-тексты:** Базовый размер `0.875rem` (`text-sm`) с интервалом `leading-relaxed` для идеальной читаемости таблиц и журналов посещаемости.

## Layout & Spacing

Вся разметка подчинена строгому **8px-шагу** координатной сетки (Scale baseline: 4px).

- **xs (4px):** Внутренние микро-зазоры между иконкой и текстом.
- **sm (8px):** Стандартные отступы по вертикали в кнопках и мелких ячейках.
- **md (16px):** Зазоры между элементами форм, внутри разделов карточек.
- **lg (24px):** Внешние паддинги карточек, крупные разрывы между разделами.
- **Планшеты и таблицы:** Ячейки посещаемости имеют фиксированную ширину `w-20` (80px), подписи учеников `w-[14rem]` с обязательным усечением длинных ФИО (`truncate`).

---

## Elevation & Depth

Дизайн основан на плоской физике элементов без ложного объема.

- **Свечение и блики:** Стандартная кнопка (`primary`) имеет внутреннийSpecular-блик на верхней границе: `inset 0px 1px 0px 0px rgba(255, 255, 255, 0.15)`.
- **Плоский ховер:** Никаких смещений `translate-y` или анимаций уменьшения `active:scale-[0.98]`. Ховер-эффекты изменяют только яркость фонового цвета (затемнение на 10% для первичных кнопок, осветление/серая подложка для второстепенных).
- **Мягкие тени:** Для карточек используется ультра-мягкая рассеянная тень `0 1px 2px rgba(16, 24, 40, 0.05)` в светлой теме и ее отсутствие (плоский контур с границей) в темной теме.

---

## Shapes

Геометрия границ жестко стандартизирована:

- **Интерактивные элементы (8px):** Все кнопки (`Button`), поля ввода (`Input`), поисковые строки и селекторы используют радиус скругления строго **8px** (класс `rounded-md` / `rounded-[8px]`).
- **Специфические мини-элементы (6px):** Размеры `xs` и `icon-xs` для кнопок округляются ровно на **6px** (`rounded-[6px]`).
- **Карточки и контейнеры (12px):** Все карточки (`Card`), модальные окна, выпадающие списки и панели используют скругление **12px** (`rounded-lg`).
- **Аватары и статусные Pill-бейджи:** Округляются по максимальной окружности (`rounded-full`).

## Components

### 1. Button (Кнопочные стандарты)
- **Primary:** Фоновый акцентный бренд-цвет с белым текстом. Внутренний полупрозрачный верхний блик.
- **Outline:** Белый или прозрачный фон, тонкая рамка `#D0D5DD` / `border-input`, текст `#344054`. На ховере — плоская подложка `bg-background-secondary`.
- **Destructive:** Насыщенный красный цвет (`#F04438`), **белый текст** (`#FFFFFF`) в обоих режимах, тонкая рамка, на ховере — легкое затемнение.
- **Ghost:** Чистый прозрачный фон без тени, текст `#475467`, на ховере — мягкий серый круглый фон.

### 2. Badge (Статусные маркеры)
- Должны иметь пастельный блеклый фон (`bg-opacity-10`) и контрастный яркий текст аналогичного оттенка. Никакого инлайнового JS или перекоса цвета.
- Не использовать скругление `rounded-[8px]`. Статусные бейджи всегда имеют форму таблетки (`rounded-full`).

### 3. Checkbox (Чекбокс контроля)
- Имеет форму квадрата со скруглением `rounded-[4px]`.
- При выборе (`checked`) заливается цветом выбранного пресета с наложением белой галочки. На ховере имеет легкую тень-кольцо (`shadow-ring`).

## Do's and Don'ts

### Do
- **Do:** Всегда использовать семантические переменные вроде `--primary-foreground` или `--destructive-foreground`, чтобы цвета автоматически адаптировались под выбранный пользователем пресет (Amethyst, Emerald, Amber, Slate).
- **Do:** Всегда принудительно накладывать на текстовые метки кнопок и инпутов семейство `font-sans` (`font-family: Inter`), гарантируя рендеринг шрифта Inter на любых мобильных клиентах.
- **Do:** Использовать только одну акцентную primary-кнопку на экране, остальные действия выполнять в виде `outline` или `secondary` кнопок.

### Don'ts
- **Don't:** Не хардкодить `text-black` или `text-slate-800` для текста, который должен инвертироваться в темном режиме. Всегда использовать Tailwind-классы с префиксом `dark:text-*` или семантический `text-foreground`.
- **Don't:** Не использовать ховер-эффекты трансформации размера / сдвиги по оси Y (`hover:-translate-y-0.5`, `active:scale-[0.97]`). Untitled UI требует плоской, надежной плоскости UI.
- **Don't:** Не мешать радиусы скруглений. Никакой кнопки со скруглением 12px или карточки со скруглением 6px.
