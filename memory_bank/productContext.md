# Product Context

## Для чего нужен проект

Сервис нужен преподавателю, который ведет занятия по разработке приложений с ИИ и хочет в одном месте:

- быстро отмечать посещаемость;
- видеть недельные нарушения нормы;
- контролировать прогресс ученических проектов на GitHub;
- получать AI-сводку по степени готовности проекта;
- отправлять уведомления через Telegram.

## Ключевые пользовательские проблемы

- посещаемость сейчас приходится считать вручную по неделям;
- проектные материалы, GitHub-репозитории и статусы прогресса разрознены;
- без централизованного AI-анализа сложно быстро увидеть проблемные проекты;
- Telegram-доставка зависит от заранее собранных `chat_id`, их нельзя надежно вывести только из username.

## Зафиксированные продуктовые решения

- MVP остается `teacher-only`;
- для Telegram доставка строится по вручную заполненным `chat_id`;
- должен существовать teacher-only экран редактирования ученика, где можно заполнять `telegram_chat_id`;
- при праздниках плановое занятие удаляется, без переноса;
- импорт XLSX не делает дедупликацию в MVP;
- прогресс на дашборде делится на четыре уровня: критический, низкий, средний, высокий.
