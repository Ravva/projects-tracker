# Appwrite Anti-Pause

## Что происходит

На `Appwrite Cloud Free` проект может автоматически перейти в паузу после `7` подряд дней без активности. В changelog Appwrite от `2026-02-20` это правило описано как требование к `development activity in the Console`, поэтому обычные API-вызовы, cron-задачи и фоновые запросы нельзя считать надежной защитой.

Источники:

- [Appwrite Changelog, 2026-02-20](https://appwrite.io/changelog/entry/2026-02-20-1)
- [Appwrite Pricing](https://appwrite.io/pricing)

## Быстрый recovery

1. Зайти в Appwrite Console.
2. Открыть проект.
3. Reactivate / Resume проект, если Appwrite уже поставил его на паузу.
4. Сразу после восстановления проверить базовые сценарии `students`, `attendance`, `projects` и Telegram webhook.

## Принятый anti-pause workflow

### Базовый шаг

Запускать:

```bash
bun run appwrite:keepalive
```

Что делает команда:

- строит прямую ссылку на текущий Appwrite project console из `APPWRITE_ENDPOINT` и `APPWRITE_PROJECT_ID`;
- открывает Console в браузере;
- пишет локальный timestamp в `.codex/appwrite-console-heartbeat.json`;
- подсказывает рекомендованную дату следующей проверки через `6` дней, чтобы не доходить до полного `7`-дневного окна.

### Что нужно сделать в Console

- убедиться, что открылся именно нужный проект;
- выполнить короткое осмысленное действие в Console, а не только открыть вкладку;
- не ждать с этим до последнего дня.

## Практические меры против повторной паузы

### 1. Операционный ритуал каждые 5-6 дней

Самый безопасный вариант для `Free` плана: раз в `5-6` дней открывать проект через `bun run appwrite:keepalive` и подтверждать активность в Console.

### 2. Внешнее напоминание

Надежнее всего завести отдельное напоминание вне приложения:

- календарь;
- системный Task Scheduler / cron;
- Telegram-напоминание самому себе;
- automation в Codex, если понадобится регулярный inbox-item.

Важно: напоминание должно помогать открыть Console, а не пытаться имитировать activity через API.

### 3. Уйти с риска Free-паузы

Если проект критичен для ежедневной работы, практичнее убрать сам источник риска:

- перейти на платный Appwrite Cloud план;
- или перенести проект на self-hosted Appwrite, если вам нужен полный контроль над жизненным циклом.

## Ограничение подхода

В этом репозитории подготовлен именно operational workflow, а не автоматический bypass. Пока Appwrite явно привязывает защиту от паузы к Console activity, честный и устойчивый способ профилактики - это регулярное живое посещение Console или отказ от Free-ограничения.
