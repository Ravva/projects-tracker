# Инструкция по использованию системы анализа логов OpenCode

## 📚 Для ученика

### Шаг 1: Работайте с OpenCode как обычно
Используйте OpenCode для разработки своего проекта. Все сессии автоматически сохраняются в логи.

### Шаг 2: Подготовьте логи к отправке
Перейдите в директорию вашего проекта:
```bash
cd путь/к/вашему/проекту
```

### Шаг 3: Получите данные для загрузки
Вам понадобятся:
- **ID проекта** - можно найти в URL страницы проекта: `https://projects-tracker-one.vercel.app/projects/[ВАШ_ID]`
- **Токен аутентификации** - получите у преподавателя или из системы

### Шаг 4: Загрузите логи
```bash
bun scripts/upload-logs.ts --project-id ВАШ_PROJECT_ID --token ВАШ_ТОКЕН
```

**Пример:**
```bash
bun scripts/upload-logs.ts --project-id 67890abc --token eyJhbGc...
```

### Шаг 5: Проверьте результат
После успешной загрузки вы увидите:
```
✅ Логи успешно загружены!
╔═══════════════════════════════════════════════════════════════╗
║  ✅ Готово! Логи загружены и готовы к анализу               ║
╚═══════════════════════════════════════════════════════════════╝
```

### 💡 Полезные опции

**Проверить что будет отправлено (без фактической отправки):**
```bash
bun scripts/upload-logs.ts --project-id ВАШ_ID --token ВАШ_ТОКЕН --dry-run
```

**Подробный вывод:**
```bash
bun scripts/upload-logs.ts --project-id ВАШ_ID --token ВАШ_ТОКЕН --verbose
```

**Справка:**
```bash
bun scripts/upload-logs.ts --help
```

### 🔒 Безопасность
Не беспокойтесь о секретах в логах - система автоматически удаляет:
- API ключи и токены
- Пароли
- Email адреса и телефоны
- Пути с вашим именем пользователя

## 👨‍🏫 Для преподавателя

### Шаг 1: Попросите ученика загрузить логи
Отправьте ученику инструкцию выше и предоставьте:
- ID его проекта (из системы)
- Токен аутентификации (если требуется)

### Шаг 2: Запустите анализ проекта
После того как ученик загрузил логи, запустите AI-анализ проекта как обычно:

1. Откройте страницу проекта: `https://projects-tracker-one.vercel.app/projects/[PROJECT_ID]`
2. Нажмите кнопку **"Запустить AI-анализ"**

### Шаг 3: Просмотрите результаты
После завершения анализа на странице проекта появится новый блок:

**🤖 AI Engineering Coach (OpenCode)**
- Оценка гигиены работы с ИИ: **X/100**
- Количество проанализированных сессий
- Список обнаруженных антипаттернов с рекомендациями

### 📊 Что показывает анализ

**Оценка (0-100 баллов):**
- 90-100: Отличная культура работы с ИИ
- 70-89: Хорошо, есть небольшие замечания
- 50-69: Удовлетворительно, нужны улучшения
- 0-49: Требуется серьезная работа над подходом

**Типы антипаттернов:**
- 🔴 Критические - серьезные проблемы в подходе
- 🟠 Высокие - важные замечания
- 🟡 Средние - рекомендации к улучшению
- 🟢 Низкие - незначительные замечания

### 🔍 Если логи не загружены
Если ученик не загрузил логи:
- Анализ проекта пройдет как обычно
- Блок OpenCode Coach просто не появится
- Это не влияет на остальные метрики проекта

### 💡 Рекомендации

**Когда запрашивать логи:**
- При подозрении на злоупотребление ИИ
- Для оценки самостоятельности работы
- При проверке финальных проектов
- Для обучения правильной работе с ИИ

**Частота загрузки:**
- Рекомендуется 1 раз в неделю для текущих проектов
- Обязательно перед финальной защитой

### 📝 Интерпретация результатов

**Хорошие признаки:**
- Высокая оценка (80+)
- Мало или нет антипаттернов
- Разнообразие типов запросов
- Итеративная работа над кодом

**Тревожные сигналы:**
- Низкая оценка (<50)
- Много критических антипаттернов
- Копирование кода без понимания
- Отсутствие рефакторинга

## ❓ Частые вопросы

**Ученик: Мои секреты будут видны?**
Нет, все чувствительные данные автоматически удаляются перед отправкой.

**Ученик: Сколько логов нужно загружать?**
Система автоматически найдет все логи в вашем проекте (максимум 150 файлов).

**Преподаватель: Как часто обновлять анализ?**
Логи хранятся 1 час, после чего нужна повторная загрузка для нового анализа.

**Преподаватель: Можно ли увидеть сами логи?**
Нет, система показывает только агрегированную статистику и антипаттерны.

## 🆘 Проблемы?

**Ошибка аутентификации:**
- Проверьте правильность токена
- Убедитесь что вы участник проекта

**Логи не найдены:**
- Проверьте наличие папок `.ai-coach/logs/` или `.opencode-logs/`
- Убедитесь что файлы имеют расширение `.json`

**Превышен лимит размера:**
- Удалите старые логи
- Загружайте только последние сессии

Подробная документация: `docs/OPENCODE_LOGS_UPLOAD.md`
