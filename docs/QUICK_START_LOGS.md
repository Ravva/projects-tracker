# Инструкция по использованию системы анализа логов OpenCode

## 📚 Для ученика

### Простой способ: Через веб-интерфейс (Рекомендуется)

**Шаг 1:** Войдите в систему через GitHub

**Шаг 2:** Откройте страницу "Ваши проекты"
- Перейдите на https://projects-tracker-one.vercel.app/my-project
- Или нажмите на свое имя в верхнем меню

**Шаг 3:** Найдите карточку "Загрузка логов OpenCode"
- Она появляется автоматически, если у вас есть текущий проект

**Шаг 4:** Загрузите логи одним из способов:

**Вариант А: Перетащите файлы**
1. Откройте папку `.ai-coach/logs/` или `.opencode-logs/` в вашем проекте
2. Выберите все `.json` файлы
3. Перетащите их в область загрузки на странице

**Вариант Б: Выберите файлы вручную**
1. Нажмите кнопку "Выбрать файлы"
2. Найдите папку с логами в вашем проекте
3. Выберите все `.json` файлы
4. Нажмите "Открыть"

**Шаг 5:** Нажмите "Загрузить логи"
- Система автоматически удалит все чувствительные данные
- Вы увидите подтверждение успешной загрузки

**Шаг 6:** Готово!
- Логи загружены и готовы к анализу
- Преподаватель может запустить AI-анализ вашего проекта

### 🔒 Безопасность
Не беспокойтесь о секретах в логах - система автоматически удаляет:
- API ключи и токены
- Пароли
- Email адреса и телефоны
- Пути с вашим именем пользователя

### ❓ Частые вопросы

**Где найти логи?**
Логи находятся в папках `.ai-coach/logs/` или `.opencode-logs/` в корне вашего проекта.

**Сколько файлов загружать?**
Загружайте все `.json` файлы из папки с логами (максимум 150 файлов).

**Как часто загружать?**
Рекомендуется загружать логи раз в неделю или перед важными проверками.

## 👨‍🏫 Для преподавателя

### Шаг 1: Попросите ученика загрузить логи
Отправьте ученику ссылку на инструкцию выше или просто скажите:
> "Зайди на страницу 'Ваши проекты' и загрузи логи OpenCode через веб-интерфейс"

### Шаг 2: Запустите анализ проекта
После того как ученик загрузил логи:

1. Откройте страницу проекта: `https://projects-tracker-one.vercel.app/projects/[PROJECT_ID]`
2. Нажмите кнопку **"Запустить AI-анализ"**

### Шаг 3: Просмотрите результаты
После завершения анализа на странице проекта появится новый блок:

**🤖 AI Engineering Coach (OpenCode)**
- Оценка гигиены работы с ИИ: **X/100**
- Количество проанализированных сессий
- Список обнаруженных антипаттернов с рекомендациями

### 📊 Что показывает анализ

**Оценка (0-100 баллов):**
- 90-100: Отличная культура работы с ИИ
- 70-89: Хорошо, есть небольшие замечания
- 50-69: Удовлетворительно, нужны улучшения
- 0-49: Требуется серьезная работа над подходом

**Типы антипаттернов:**
- 🔴 Критические - серьезные проблемы в подходе
- 🟠 Высокие - важные замечания
- 🟡 Средние - рекомендации к улучшению
- 🟢 Низкие - незначительные замечания

### 🔍 Если логи не загружены
Если ученик не загрузил логи:
- Анализ проекта пройдет как обычно
- Блок OpenCode Coach просто не появится
- Это не влияет на остальные метрики проекта

### 💡 Рекомендации

**Когда запрашивать логи:**
- При подозрении на злоупотребление ИИ
- Для оценки самостоятельности работы
- При проверке финальных проектов
- Для обучения правильной работе с ИИ

**Частота загрузки:**
- Рекомендуется 1 раз в неделю для текущих проектов
- Обязательно перед финальной защитой

### 📝 Интерпретация результатов

**Хорошие признаки:**
- Высокая оценка (80+)
- Мало или нет антипаттернов
- Разнообразие типов запросов
- Итеративная работа над кодом

**Тревожные сигналы:**
- Низкая оценка (<50)
- Много критических антипаттернов
- Копирование кода без понимания
- Отсутствие рефакторинга

## 🆘 Проблемы?

**Не вижу карточку загрузки логов:**
- Убедитесь что у вас есть текущий проект
- Обновите страницу

**Ошибка при загрузке:**
- Проверьте что файлы имеют расширение `.json`
- Убедитесь что размер файлов не превышает 10MB каждый
- Попробуйте загрузить меньше файлов за раз

**Логи не найдены в проекте:**
- Проверьте наличие папок `.ai-coach/logs/` или `.opencode-logs/`
- Убедитесь что вы работали с OpenCode в этом проекте

Подробная техническая документация: `docs/OPENCODE_LOGS_UPLOAD.md`
